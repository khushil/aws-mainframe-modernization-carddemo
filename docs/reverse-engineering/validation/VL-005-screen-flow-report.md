### MJ-001: BMS Copybooks Not Referenced

| Field | Value |

| **Severity** | Major |
| **Category** | Completeness |
| **Location** | Section 10 "Source References" (line 1047-1057) |

**Description:** The document references `app/cpy/` for copybooks but never mentions the `app/cpy-bms/` directory, which contains 17 BMS-generated copybooks (one per mapset). **Impact:** Incomplete source reference inventory. Modernization teams need to know about BMS copybooks to map screen fields to COBOL data items.

**Remediation:** Add `app/cpy-bms/` to the source references table and note that each BMS mapset has a corresponding symbolic map copybook.

### MN-001: COUSR01 PF12 Display vs Implementation Discrepancy Not Noted

| Field | Value |
|-------|-------|
| **Severity** | Minor |
| **Category** | Factual Accuracy |
| **Location** | Section 4.2 "Screen-Specific PF Key Matrix" (line 547) |

**Description:** The document correctly shows PF12=Exit for COUSR01, matching the BMS footer display. However, `COUSR01C.cbl` has NO DFHPF12 handler -- pressing PF12 falls through to `WHEN OTHER` which displays "Invalid key pressed." The document should note this implementation discrepancy.

**Evidence:**
- `app/bms/COUSR01.bms:159`: `INITIAL='ENTER=Add User  F3=Back  F4=Clear  F12=Exit'`
- `app/cbl/COUSR01C.cbl`: No `DFHPF12` match found (grep returns empty)
- PF12 hits `WHEN OTHER` branch, displaying invalid key message

**Impact:** Minor -- the document reflects the BMS display which is the user-visible behavior, but the actual runtime behavior differs. Could cause confusion during modernization if the new implementation honors PF12=Exit.

**Remediation:** Add a footnote or annotation noting that COUSR01 BMS displays F12=Exit but the COBOL program does not implement a PF12 handler.
